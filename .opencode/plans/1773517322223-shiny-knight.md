# Fix: DoneView stats animate from 0 every time

## Problem

Every time `DoneView` appears, the stat counters (Streak, Read, Written) animate from 0 to their actual values. This creates a satisfying moment after completing a day's writing, but feels weird in these cases:

- Returning from bonus read or free write back to DoneView
- App launch when the day is already completed
- App returning from background while on DoneView

The animation should only play once: right after the user completes their daily writing.

## Root Cause

- `DoneView` line 13: `@State private var animateStats = false` resets to `false` every time the view is recreated
- `DoneView` line 127: `.onAppear` always triggers `animateStats = true` after 0.6s
- `AnimatedStatColumn` line 11: `@State private var displayValue: Int = 0` always starts at 0
- Since `DoneView` is removed/re-added to the view hierarchy on every phase change, `@State` resets and the animation replays

## Solution

Thread a `shouldAnimateStats` flag from `RootView` through `DoneView` to `AnimatedStatColumn`. Only set it to `true` when transitioning from `.write` -> `.done` (a fresh completion).

## Files to Modify

### 1. `apps/ios/betterwriter/betterwriter/Views/RootView.swift`

- Add `@State private var animateDoneStats = false` (after line 15)
- In `WriteView`'s `onComplete` (line 41): set `animateDoneStats = true` before `advanceState()`
- In `DoneView`'s `onBonusRead` closure (line 54): set `animateDoneStats = false` before phase change
- In `DoneView`'s `onFreeWrite` closure (line 57): set `animateDoneStats = false` before phase change
- Pass `shouldAnimateStats: animateDoneStats` to `DoneView` (line 52)

### 2. `apps/ios/betterwriter/betterwriter/Views/DoneView.swift`

- Add `let shouldAnimateStats: Bool` parameter (after line 7)
- Modify `.onAppear` (lines 125-129): only use the 0.6s delay when `shouldAnimateStats` is true; otherwise set `animateStats = true` immediately (so `onChange` fires but AnimatedStatColumn skips animation)
- Pass `skipAnimation: !shouldAnimateStats` to each `AnimatedStatColumn` (lines 41-58)

### 3. `apps/ios/betterwriter/betterwriter/Views/Components/AnimatedStatColumn.swift`

- Add `let skipAnimation: Bool` property
- Write explicit `init` that initializes `_displayValue = State(initialValue: skipAnimation ? targetValue : 0)` -- this prevents the "flash of 0" entirely
- In `.symbolEffect(.bounce, value:)` (line 19): use `skipAnimation ? false : animate` so icons don't bounce on non-animated appearances
- In `.onChange(of: animate)` (lines 36-43): when `skipAnimation` is true, set `displayValue = targetValue` without `withAnimation`

## Behavior After Fix

| Navigation to DoneView | `animateDoneStats` | Animation |
|---|---|---|
| Complete daily writing (`.write` -> `.done`) | `true` | Stats count up from 0 with spring + bounce |
| Return from bonus read | `false` | Stats show actual values instantly |
| Return from free write | `false` | Stats show actual values instantly |
| App launch (day already done) | `false` (default) | Stats show actual values instantly |
| App foreground (already on DoneView) | n/a | DoneView not recreated, no change |

## Verification

1. Build: `bundle exec fastlane build` from `apps/ios`
2. Manual test flow:
   - Complete daily read+write -> stats should animate from 0 (with delay)
   - Tap "Read something" -> complete/back -> stats should show instantly (no animation)
   - Tap "Write something" -> complete/back -> stats should show instantly (no animation)
   - Kill app, relaunch when day is done -> stats should show instantly
   - Background and foreground the app -> stats should not re-animate
