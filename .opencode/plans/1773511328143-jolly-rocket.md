# Fix: Infinite loading after app reinstall

## Previous Fix (completed)
Server-side FK constraint error resolved. PUT /api/entries now returns 401 when user doesn't exist.

## Current Problem
App gets stuck on `.loading` screen indefinitely after reinstall.

### Root Cause
`DayEngine.resolveCurrentPhase()` returns `.loading` **only** when `profile` is nil.

The `.task` + async `ensureProfile()` change introduced a timing gap:
1. `.task` fires, `ensureProfile()` creates a profile in SwiftData then **suspends** at `await authenticate()`
2. During suspension, `@Query profiles` may or may not pick up the insert (SwiftData observation timing is non-deterministic)
3. After `.task` completes, `advanceState(profileOverride:)` correctly transitions to `.read`
4. But subsequent `advanceState()` calls **without** `profileOverride` (from `onChange(of: scenePhase)`, child view callbacks) use `profiles.first` from `@Query` — which may still be nil
5. `resolveCurrentPhase(profile: nil, ...)` returns `.loading` → app snaps back to loading

The core issue: `advanceState()` depends on `@Query profiles.first` for the profile, but `@Query` updates are asynchronous and unreliable during the `.task` suspension window.

## Fix

### iOS: Store profile in `@State` as fallback for `@Query`

**File:** `apps/ios/betterwriter/betterwriter/Views/RootView.swift`

**Changes:**

1. Add a `@State` property to hold the resolved profile independently of `@Query`:
```swift
@State private var resolvedProfile: UserProfile?
```

2. Change `.task` to also set `resolvedProfile`:
```swift
.task {
    let profile = await ensureProfile()
    resolvedProfile = profile
    advanceState(profileOverride: profile)
}
```

3. Update `advanceState()` to use `resolvedProfile` as fallback:
```swift
private func advanceState(profileOverride: UserProfile? = nil) {
    if case .freeWrite = currentPhase { return }
    if case .bonusRead = currentPhase { return }

    let effectiveProfile = profileOverride ?? resolvedProfile ?? profile

    let resolved = DayEngine.resolveCurrentPhase(
        profile: effectiveProfile,
        entries: Array(entries)
    )
    if resolved != currentPhase {
        currentPhase = resolved
    }
}
```

This ensures every `advanceState()` call — whether from `.task`, `onChange`, child callbacks, or foreground events — always has access to a non-nil profile once it's been created.
