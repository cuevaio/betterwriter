# Fix: FOREIGN KEY constraint on entries after app reinstall

## Root Cause

The iOS Keychain **persists across app reinstalls**. When the user:
1. Deleted all users/entries from Turso DB
2. Reinstalled the iOS app

The old device UUID and JWT remained in Keychain. On fresh launch:
- `ensureProfile()` in `RootView.swift:117-139` fires `POST /api/auth` (which creates the user on the server) in a **fire-and-forget background `Task`**
- `advanceState()` runs **immediately** (line 79), before auth completes
- The app enters `.read(dayIndex: 0)`, starts streaming readings
- Server validates the JWT (still valid, 30-day expiry), dispatches reading generation
- Reading generation and `PUT /api/entries` fail with `SQLITE_CONSTRAINT: FOREIGN KEY` because the user row doesn't exist yet

## Fix

### 1. iOS: Await auth before advancing state (primary fix)

**File:** `apps/ios/betterwriter/betterwriter/Views/RootView.swift`

Change `ensureProfile()` to be async so it can await the `POST /api/auth` call for new profiles. Update `onAppear` to use an async Task.

**Before:**
```swift
.onAppear {
    let profile = ensureProfile()
    advanceState(profileOverride: profile)
}
```

**After:**
```swift
.onAppear {
    Task { @MainActor in
        let profile = await ensureProfile()
        advanceState(profileOverride: profile)
    }
}
```

Change `ensureProfile()` from sync to async:
```swift
private func ensureProfile() async -> UserProfile {
    let descriptor = FetchDescriptor<UserProfile>()
    if let existing = try? modelContext.fetch(descriptor).first {
        return existing
    }
    let newProfile = UserProfile()
    modelContext.insert(newProfile)
    try? modelContext.save()
    // AWAIT auth so user exists on server before any other API calls
    do {
        _ = try await APIClient.shared.authenticate(installDate: newProfile.installDate)
    } catch {
        print("Failed to authenticate with server: \(error)")
    }
    return newProfile
}
```

Also update the `onChange(of: profiles)` call (line 84) to match the async pattern.

### 2. Server: User existence check in PUT /api/entries (defense in depth)

**File:** `apps/web/app/api/entries/route.ts`

After `requireUserId()` (line 132), before the upsert, verify the user exists in the DB. If not, return 401 so the iOS 401-retry mechanism re-authenticates (creating the user) and replays the request.

```typescript
// After line 132
const userExists = await db
  .select({ id: users.id })
  .from(users)
  .where(eq(users.id, userId))
  .limit(1);

if (userExists.length === 0) {
  return NextResponse.json(
    { error: "User not found" },
    { status: 401 }
  );
}
```

Import `users` from `@/lib/db/schema`.
