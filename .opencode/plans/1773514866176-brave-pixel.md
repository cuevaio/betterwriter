# Plan: Elevate iOS App Visual Design

**Goal:** Transform betterwriter's iOS app from "plain minimalism" to "deliberately elegant minimalism" — like upgrading from a cheap paperback to a beautifully typeset hardcover. Keep the literary identity, add premium polish.

**Scope:** 6 new files, ~20 modified files across `apps/ios/betterwriter/betterwriter/`

---

## Phase 0: Foundation — Shared Components & Cleanup

### 0A. Extract Duplicated Subviews

**Create 3 new files in `Views/Components/`:**

| File | Replaces | Used By |
|---|---|---|
| `WQLoadingView.swift` | Inline `loadingView` (3 copies) | ReadView, BonusReadView, WriteView |
| `WQErrorView.swift` | Inline `errorView` (3 copies) | ReadView, BonusReadView, WriteView |
| `WQMarkdownContent.swift` | Inline `markdownContent` (3 copies) | ReadView, BonusReadView, ReadingLogView |

- `WQErrorView` adds an SF Symbol (`exclamationmark.triangle`) above the message
- `WQLoadingView` centers vertically with `frame(maxHeight: .infinity)` instead of `Spacer(minLength: 200)` hacks

### 0B. Wire Up TypewriterAnimator

The `TypewriterAnimator` utility class exists but is not used by any views — all 3 views (ReadView, WriteView, BonusReadView) copy-paste the `enqueueTypewriter` logic inline. Refactor each to use a `@State private var typewriter = TypewriterAnimator()` instance.

### 0C. Eliminate Magic Numbers

Add to `Spacing.swift`:
- `centeredMinSpacer: CGFloat = 120`
- `textEditorMinHeight: CGFloat = 300`
- `focusDelayNs: UInt64 = 600_000_000`

Replace all hardcoded values in ReadView, BonusReadView, WriteView, FreeWriteView.

**Files modified:** `Spacing.swift`, `ReadView.swift`, `BonusReadView.swift`, `WriteView.swift`, `FreeWriteView.swift`

## Phase 1: Color & Contrast

**Problem:** Pure `#000000` on `#FFFFFF` (21:1 contrast) causes eye strain during extended reading.

### 1A. Update `AppBackground` Asset Color

| Mode | Current | New |
|---|---|---|
| Light | `#FFFFFF` pure white | `#F9F7F4` warm ivory |
| Dark | `#000000` pure black | `#121110` warm near-black |

### 1B. Create New `AppForeground` Asset Color

| Mode | Value |
|---|---|
| Light | `#191614` warm off-black |
| Dark | `#EAE8E5` warm off-white |

### 1C. Overhaul `WQColor.swift`

- Derive all colors from `Color("AppForeground")` instead of `Color.primary`
- Add `tertiary` (35% opacity) and `divider` (10% opacity) tokens
- Make activity chart monochrome (remove green) — all levels use foreground at varying opacity
- Unify compact/full chart colors (fixes bug #7)

**Files modified:** `AppBackground.colorset/Contents.json`, `WQColor.swift`
**Files created:** `AppForeground.colorset/Contents.json`

---

## Phase 2: Typography Refinements

Changes in `Typography.swift`:
- `sansButton`: `.semibold` -> `.medium` (less heavy in uppercase)
- `statNumber`: `.rounded .bold` -> `.serif .regular` (literary feel, not fitness-app)
- Add `sansFootnote`, `readingLineSpacing` (7), `promptLineSpacing` (5), `editorLineSpacing` (6)

Button tracking: `2` -> `1.5` (less shouty)
Stat label tracking: add `1` (refined uppercase)

**Files modified:** `Typography.swift`, all views that hardcode `.lineSpacing(6)` or `.lineSpacing(4)`

---

## Phase 3: Button Evolution

### 3A. Overhaul `WQOutlinedButtonStyle`

- Add `4pt` corner radius with `.continuous` superellipse
- Spring press animation: scale `0.975` + opacity `0.85` (spring response 0.25s, damping 0.7)
- New `isFilled: Bool` parameter for primary CTAs (inverted colors)
- Border weight: `1.5` -> `1` for outlined variant
- "DONE READING/WRITING" buttons become filled; "Read/Write something" stay outlined

### 3B. Add Haptic Feedback

**Create `Utilities/Haptics.swift`** with static `light()`, `medium()`, `success()`, `selection()`.

Integration:
- `completeReading()`: `Haptics.medium()`
- `completeWriting()` (day completion): `Haptics.success()`
- DoneView action buttons: `Haptics.light()`

**Files modified:** `WQOutlinedButtonStyle.swift`, `ReadView.swift`, `WriteView.swift`, `BonusReadView.swift`, `FreeWriteView.swift`, `DoneView.swift`
**Files created:** `Utilities/Haptics.swift`

---

## Phase 4: DoneView Glow-Up (Biggest Visual Impact)

### 4A. DoneView Improvements

- Wrap in `ScrollView` (fixes bug #2 - clips on small screens)
- New user empty state: "Welcome" hero + "Complete your first reading and writing to see your stats here"
- SF Symbols on action buttons: `Label("Read something", systemImage: "book.pages")`, `Label("Write something", systemImage: "pencil.line")`
- Log links get `list.bullet` symbol
- Button spacing: `Spacing.l` (24) -> `Spacing.m` (16) between action buttons

### 4B. Animated Stat Counters

**Create `Views/Components/AnimatedStatColumn.swift`:**
- Numbers animate from 0 to target with `spring(response: 0.8, dampingFraction: 0.8)`
- `.contentTransition(.numericText())` for digit morphing
- SF Symbol above each stat (`flame`, `book`, `pencil.line`) with `.symbolEffect(.bounce)` on appear
- 0.6s delay after appear for staggered entry

Replaces `StatColumnView` in DoneView (keep StatColumnView for potential reuse).

**Files modified:** `DoneView.swift`
**Files created:** `Views/Components/AnimatedStatColumn.swift`

## Phase 6: Micro-Interactions & Polish

### 6A. Phase Transitions
- `RootView` animation: `.easeInOut(duration: 0.5)` -> `.spring(response: 0.5, dampingFraction: 0.85)`
- Read/Write: horizontal slide (forward progression)
- BonusRead/FreeWrite: slide up from bottom (overlay mental model)
- DoneView: gentle fade

### 6B. Fix Double Wordmark (Bug #1)
Remove `BrandWordmarkView()` from `loadingView` in `RootView.swift`. The safeAreaInset wordmark is always visible.

### 6C. Loading State Polish
Add subtle opacity pulse to loading captions: `easeInOut(1.5s).repeatForever`

### 6D. Consistent Bottom Bar Material
Standardize all bottom bars to: `.background(WQColor.background.opacity(0.9).background(.ultraThinMaterial))`

**Files modified:** `RootView.swift`, `ReadView.swift`, `WriteView.swift`, `BonusReadView.swift`, `FreeWriteView.swift`
