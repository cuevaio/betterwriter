# Plan: Configure InjectionIII + Inject Hot Reload for iOS App

## Goal
Add the [Inject](https://github.com/krzysztofzablocki/Inject) SPM package to the betterwriter iOS app so that view changes are reflected instantly in the running simulator via InjectionIII, without a full rebuild.

## Key Insight
Inject is a NO-OP in release builds. `@ObserveInjection` and `.enableInjection()` compile to inlined identity operations when `DEBUG` is not defined. No `#if DEBUG` wrapping needed — the code stays permanently in view files.

### Standard pattern (example: RootView.swift)
```swift
import Inject          // <-- add
import SwiftData
import SwiftUI

struct RootView: View {
  @ObserveInjection var inject  // <-- add
  @Environment(\.scenePhase) private var scenePhase
  ...
  var body: some View {
    ZStack { ... }
      .onChange(of: scenePhase) { ... }
      .enableInjection()  // <-- add as last modifier
  }
}
```


## Step 4: Resolve Packages and Verify

1. Run `xcodebuild -resolvePackageDependencies` to download Inject
2. Run Debug simulator build to confirm compilation:
   ```
   xcodebuild build \
     -project apps/ios/betterwriter/betterwriter.xcodeproj \
     -scheme betterwriter \
     -destination 'platform=iOS Simulator,name=iPhone 16' \
     -configuration Debug
   ```
3. Verify Release still builds (Inject compiles to no-ops):
   ```
   xcodebuild build \
     -project apps/ios/betterwriter/betterwriter.xcodeproj \
     -scheme betterwriter \
     -destination 'generic/platform=iOS' \
     -configuration Release \
     CODE_SIGNING_ALLOWED=NO
   ```

## Prerequisites (User Action)
- Install [InjectionIII.app](https://github.com/johnno1962/InjectionIII/releases) in `/Applications/`
- Run InjectionIII, point it at the project directory
- Build and run the app in Xcode simulator
- Edit any view file + save → changes appear live
