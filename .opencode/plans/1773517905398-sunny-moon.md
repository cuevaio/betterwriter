# Fix: iOS app stuck on "Finding and streaming today's reading..."

## Problem
TestFlight users open the iOS app and it stays on "Finding and streaming today's reading..." indefinitely. Vercel logs only show `POST /api/sync` (200) — no `/api/readings/generate/stream` requests appear.

## Root Cause
Two compounding issues:

### 1. Server: No user auto-creation outside `/api/auth`
If a user's DB row is missing (DB reset, migration, etc.) while the iOS app still has a valid JWT in Keychain:
- `/api/sync` returns 200 with `{ user: null, entries: [], currentDayIndex: 0 }` — the `UPDATE users` silently affects 0 rows
- `/api/readings/generate/stream` background worker emits an SSE `"error"` event with `"User not found"` (line 97-104)
- Entry upserts would fail with FK constraint violation if entries are in the sync payload

### 2. Client: PrefetchStore not reset on foreground after failure
`PrefetchStore` only resets when `currentDay != lastPrefetchedDayIndex` (`RootView.swift:135`). For a new user where both are `0`, the failed prefetch is never retried. `ReadView.task` (line 47) runs once and won't re-fire on foreground.

---

## Changes

### 1. New file: `apps/web/lib/db/ensure-user.ts`
Create a shared `ensureUser(userId)` helper:
- `SELECT` first (fast path for existing users)
- If missing: `INSERT ... ON CONFLICT DO NOTHING` + fallback `SELECT`
- Handles concurrent race conditions safely
- Sets minimal required fields: `id`, `createdAt`, `installDate` (all NOT NULL)

### 2. Edit: `apps/web/app/api/sync/route.ts`
- Import `ensureUser`
- Call `ensureUser(userId)` after `requireUserId` (before any DB writes)
- Use fallback in response: `user: updatedUser[0] || userRow` (never returns `null`)

### 3. Edit: `apps/web/app/api/readings/generate/stream/route.ts`
- Import `ensureUser`
- In `runReadingGeneration` (line 88-105): replace `userRows` query + "User not found" error path with `ensureUser(userId)` call in the parallel `Promise.all`

### 4. Edit: `apps/web/app/api/prompts/generate/stream/route.ts`
- Import `ensureUser`
- In `runPromptGeneration` (line 59-86): replace `userRows` query + "User not found" error path with `ensureUser(userId)` call in the parallel `Promise.all`

### 5. Edit: `apps/ios/.../Views/RootView.swift` (lines 130-151)
- In `.onChange(of: scenePhase)`, add: if `PrefetchStore.shared.reading` is `.failed`, reset and re-prefetch
- This ensures failed prefetches are retried on foreground even when the day hasn't changed

### 6. Edit: `apps/ios/.../Views/ReadView.swift` (after line 49)
- Add `.onReceive(NotificationCenter.default.publisher(for: UIApplication.didBecomeActiveNotification))` that calls `loadReading()` if `errorMessage != nil`
- Ensures ReadView retries on foreground when showing an error state
