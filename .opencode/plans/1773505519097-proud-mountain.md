# Plan: App Store Screenshots Generator

## Overview

Create a Next.js page at `/screenshots` that renders 6 iPhone App Store screenshot slides for the betterwriter iOS app, with export to PNG at Apple's required resolutions via `html-to-image`.

## Prerequisites

User must capture these device screenshots and place them in `apps/web/public/screenshots/`:
- `read.png` — ReadView (serif text mid-stream, book-page layout)
- `write.png` — WriteView (prompt visible + some written text + word count)
- `write-empty.png` — WriteView (prompt visible + empty editor with placeholder)
- `done.png` — DoneView (stats + activity chart with some data)

The generator will show gray placeholders until real screenshots are added.

## Slide Deck (6 slides)

| # | Role | BG | Layout | Copy | Screenshot |
|---|------|-----|--------|------|------------|
| 1 | Hero | White | Icon centered top, phone centered bottom | `DAILY PRACTICE` / Read. Remember. Write. | read.png |
| 2 | Differentiator | **Black** | Phone right-offset, caption left-aligned | `THE METHOD` / Two days later, write what stuck. | write-empty.png |
| 3 | Feature: Read | White | Phone center-left, caption upper-right | `READING` / No scrolling. No choosing. Just read. | read.png |

| 6 | Closing | **Black** | No phone — icon + wordmark only | Build a mind that writes. | none |

### 3. Copy assets

### 4. Build `apps/web/app/screenshots/page.tsx`

Single `"use client"` file containing:

**Constants:**
- Canvas: `IPHONE_W = 1320`, `IPHONE_H = 2868`
- Export sizes: 4 iPhone sizes (6.9", 6.5", 6.3", 6.1")
- Mockup measurements: `MK_W=1022`, `MK_H=2082`, screen offsets
- Colors hardcoded (not CSS vars — required for html-to-image export)

**Components (all inline styles, no Tailwind):**
- `Phone` — mockup.png overlay with screenshot clipped inside
- `Caption` — category label + headline, font sizes relative to canvasW
- `BWIcon` — inline SVG of the BW app icon (hardcoded black/white, not CSS vars)
- `Slide1` through `Slide6` — one component per slide
- `ScreenshotPreview` — ResizeObserver-based scaling for preview grid
- `ScreenshotsPage` — grid layout + size picker + export buttons + offscreen container

**Slide layouts (matching the plan above):**
- Slides 1, 3, 5: white bg (`#ffffff`), black text
- Slides 2, 4, 6: black bg (`#000000`), white text
- No gradients, no blobs — monochrome brand. Whitespace is the decoration
- Optional thin horizontal rules on editorial slides

**Export logic:**
- Offscreen container at `position: absolute; left: -9999px`
- Double-call `toPng()` trick (first warms fonts, second is clean)
- Filenames: `01-hero-1320x2868.png`, `02-method-1320x2868.png`, etc.
- Size dropdown lets user pick target resolution
- "Export All" downloads all 6 at selected size

**Key technical decisions:**
- All inline styles (matches codebase — no Tailwind)
- Hardcoded colors (CSS vars don't survive html-to-image export)
- Georgia serif + system sans-serif (no custom font loading needed)
- Gray placeholder rectangles when screenshot PNGs are missing

6. Click export — downloads PNG at selected resolution
