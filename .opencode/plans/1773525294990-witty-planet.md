# Plan: Store memories for free writes (extra writings)

## Problem

Free writes (dayIndex >= 200,000) submitted via `PUT /api/entries` never store memories in Mem0. Normal day writings go through `POST /api/user-input` which calls `addUserInputMemory()`. Free writes bypass this entirely, so the AI never learns from free write content.

## Changes

**Single file:** `apps/web/app/api/entries/route.ts`

### 1. Add import (after line 3)

```typescript
import { addUserInputMemory } from "@/lib/ai/mem0";
```

Place after the existing `@/lib/ai/durable-stream` import to maintain alphabetical order.

### 2. Add fire-and-forget memory call (after line 194, before the return)

```typescript
// Store free write text as memories in Mem0 (fire-and-forget)
if (
  allowedFields.isFreeWrite === true &&
  allowedFields.writingCompleted === true &&
  allowedFields.writingText &&
  allowedFields.writingText.trim()
) {
  addUserInputMemory(userId, allowedFields.writingText).catch((err) => {
    console.error("Failed to add free write memory:", err);
  });
}
```

**Guard conditions:**
- `isFreeWrite === true` — only free writes; normal entries use `POST /api/user-input`
- `writingCompleted === true` — only final submissions, not draft saves
- `writingText` is non-empty — skip blank/whitespace text

**Pattern:** identical fire-and-forget with `.catch()` as `POST /api/user-input` line 44.

## Not changed

- `POST /api/user-input` — already handles normal writings
- `POST /api/sync` — out of scope (bulk sync shouldn't re-process)
- `lib/ai/mem0.ts` — no changes needed; handles edge cases internally
