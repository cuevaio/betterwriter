# Investigate "Done Reading" button not working

## What is happening

The issue is most likely a **phase refresh timing bug** in iOS `RootView`:

- In `ReadView.completeReading()`, tapping `DONE READING` updates the local `DayEntry` (`readingCompleted = true`) and immediately calls `onComplete()`.
- `onComplete()` calls `advanceState()` in `RootView`.
- `advanceState()` resolves the next phase from `@Query entries`.
- If `@Query entries` has not propagated the just-saved `readingCompleted` change yet, `DayEngine.resolveCurrentPhase(...)` still returns `.read(dayIndex: ...)`.
- `RootView` only recomputes on `.task`, profile changes, or scene active; it does **not** recompute when `entries` changes.
- Result: tap can appear to do nothing (stays on reading screen) until another lifecycle event triggers refresh.

This same timing pattern also exists for `WriteView.completeWriting()` and can cause similar stale-transition behavior.

## Files involved

- `apps/ios/betterwriter/betterwriter/Views/RootView.swift`
- `apps/ios/betterwriter/betterwriter/Views/ReadView.swift`
- `apps/ios/betterwriter/betterwriter/Views/WriteView.swift`
- `apps/ios/betterwriter/betterwriter/Services/DayEngine.swift`
- `apps/ios/betterwriter/betterwriter/Views/BonusReadView.swift` (related but less likely the main failing path)

## Recommended implementation

1. Update `RootView` to react to `entries` changes:
   - Add `.onChange(of: entries)` and call `advanceState()`.
   - Keep existing guards in `advanceState()` for `.bonusRead` / `.freeWrite` overlays.

2. Keep optimistic UX in `ReadView.completeReading()` (local save + immediate `onComplete()`), but rely on new `entries` change observer as the deterministic fallback when `@Query` updates asynchronously.

3. (Optional hardening) In `ReadView.completeReading()`, after local save, schedule a short delayed `onComplete()` retry only if still on read phase (not required if step 1 is sufficient).

4. Confirm no regressions in bonus flow:
   - `BonusReadView` should still return to `.done(dayIndex: ...)` after completion.

## Verification plan

1. Build iOS app:
   - `cd apps/ios && bundle exec fastlane build`

2. Manual daily reading flow test:
   - Start at `.read(dayIndex: N)`.
   - Finish stream so `DONE READING` appears.
   - Tap `DONE READING` once.
   - Expect immediate transition to `.write(dayIndex: N, ...)` every time.

3. Manual write completion sanity check:
   - Complete writing and tap `DONE WRITING`.
   - Expect transition to `.done(dayIndex: N)` without requiring background/foreground.

4. Bonus reading sanity check:
   - From done screen open bonus reading, complete it, tap `DONE READING`.
   - Expect return to `.done(dayIndex: N)`.

5. Foreground/background edge case:
   - During reading session, background app and return.
   - Ensure final completion still transitions correctly.
