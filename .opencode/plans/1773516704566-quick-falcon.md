## Gaps

### 1. Hard timeout mismatch (Step 2)
- **Plan:** 30s hard timeout in `createInlineSSEResponse`
- **Actual:** 120s (2 min) at `apps/web/lib/api/durable-sse.ts:153`
- **Impact:** Low — 120s is arguably more robust for slow connections. This was likely a deliberate adjustment.

### 2. Missing day-advance re-prefetch (Step 7)
- **Plan:** In `RootView.swift` `onChange(of: scenePhase)` when `.active`, reset and re-prefetch if the day has advanced.
- **Actual:** The `onChange` handler at `apps/ios/betterwriter/betterwriter/Views/RootView.swift:119-131` only calls `advanceState()` and starts a sync task. It does **not** check if the calendar day changed or call `PrefetchStore.shared.reset()` + `prefetch()`.
- **Impact:** Medium — if the user leaves the app open overnight, returning the next day won't pre-fetch new content until they fully restart the app.

### 3. Stale comment (cosmetic)
- `apps/web/lib/ai/durable-stream.ts:272` comment still says "10+ min" but `STALE_RUNNING_MS` is correctly set to 2 min.

## Recommendation

The only functional gap worth addressing is **Step 7** (day-advance re-prefetch in `RootView.swift`). The timeout deviation and stale comment are minor.

### Fix for Step 7
In `RootView.swift`, inside the `onChange(of: scenePhase)` handler when phase is `.active`:
1. Compare the current day index against the last-known day index
2. If it has advanced, call `PrefetchStore.shared.reset()` then `PrefetchStore.shared.prefetch()`

- `apps/ios/betterwriter/betterwriter/Views/RootView.swift` — add day-advance detection + re-prefetch
