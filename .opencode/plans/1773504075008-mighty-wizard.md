# Plan: Fix Production API URL in iOS App

## Context

- Production backend is deployed at **`https://betterwriter.vercel.app`** (no custom domain yet).
- The iOS `APIClient.swift` currently hardcodes `https://betterwriter.app` as the release URL — a domain that doesn't exist yet.
- The `#if DEBUG` / `#else` compile-time switch is the right pattern for this project; no change to the architecture is needed.
- The fix is a one-line string change.

## Problem

```swift
// apps/ios/betterwriter/betterwriter/Services/APIClient.swift  lines 8–12
#if DEBUG
  private let baseURL = "http://localhost:3000"
#else
  private let baseURL = "https://betterwriter.app"   // ← wrong / non-existent domain
#endif
```

Release builds currently point at `betterwriter.app`, which has no DNS record, so all production API calls from the iOS app fail.

## Solution

Change the release URL to `https://betterwriter.vercel.app`. No architectural change; the existing `#if DEBUG` switch is the correct, simple approach for two environments.

```swift
#if DEBUG
  private let baseURL = "http://localhost:3000"
#else
  private let baseURL = "https://betterwriter.vercel.app"
#endif
```

When a custom domain is acquired later, this single line is the only thing that needs to change.
