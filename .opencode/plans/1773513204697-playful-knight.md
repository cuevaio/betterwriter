# SwiftUI Code Review & Optimization Plan (COMPLETED)

Analysis of `apps/ios/betterwriter/betterwriter/` (29 Swift files) against SwiftUI best practices from the `swiftui-expert-skill`.

---

## Phase 1: Thread Safety & Correctness Fixes

### 1.1 Make `StreamSessionStore` thread-safe
**File:** `Services/StreamSessionStore.swift`
- `StreamSessionStore` is a plain `class` with mutable `Set<String>` state
- Currently called only from `@MainActor` views, but nothing enforces this
- **Fix:** Add `@MainActor` annotation to the class
- Risk: Low (all callers are already `@MainActor`)

### 1.2 Add `onTermination` handler to `AsyncThrowingStream` in APIClient
**File:** `Services/APIClient.swift` (lines ~143, ~337)
- The `Task` spawned inside `AsyncThrowingStream` is never cancelled when the consumer stops iterating
- **Fix:** Set `continuation.onTermination = { _ in task.cancel() }` for both `streamReading()` and `streamPrompt()`
- Risk: Low

### 1.3 Fix "Pull to retry" with no retry mechanism in WriteView
**File:** `Views/WriteView.swift` (line ~280)
- Error message says "Pull to retry" but there is no `.refreshable` modifier
- **Fix:** Change the message to match actual behavior (show a "Try again" button like `ReadView` does), or add `.refreshable`
- Risk: Low

## Phase 2: Performance Optimizations

### 2.1 Debounce `saveDraft()` on keystroke
**Files:** `Views/WriteView.swift` (lines ~107-109, ~287-292), `Views/FreeWriteView.swift` (lines ~75-77, ~127-131)
- Currently calls `modelContext.save()` (SQLite write) on every keystroke via `onChange(of: userText)`
- **Fix:** Debounce saves with a `Task.sleep` pattern (e.g., 500ms delay, cancel previous task on new keystroke). Keep an immediate save on `completeWriting()` and `onBack`.
- Risk: Medium (must ensure data is saved before view disappears)

### 2.2 Cache `Array(entries)` in DoneView and StatsView
**Files:** `Views/DoneView.swift` (lines ~14-24, ~48), `Views/StatsView.swift` (lines ~9-17, ~34)
- Multiple computed properties each call `Array(entries)` + iterate the full list
- Up to 4-5 array copies per body evaluation
- **Fix:** Compute `Array(entries)` once in body and pass to child helpers, or use `.onChange(of: entries)` to cache aggregated stats in `@State`
- Risk: Low

### 2.3 Cache markdown `AttributedString` parsing
**Files:** `Views/ReadView.swift`, `Views/BonusReadView.swift`, `Views/Components/ReadingLogView.swift`
- `markdownAttributedString()` runs on every body evaluation
- For long reading content this can cause frame drops
- **Fix:** Store the parsed `AttributedString` in `@State` and update via `.onChange(of:)` or `.task(id:)` when the source text changes
- Risk: Low

### 2.4 Optimize `wordCount` computation
**Files:** `Views/WriteView.swift` (lines ~28-32), `Views/FreeWriteView.swift` (lines ~17-21)
- String split + filter runs at least twice per keystroke (body + saveDraft)
- **Fix:** Store `wordCount` in `@State`, update in `.onChange(of: userText)` (where saveDraft already runs)
- Risk: Low

## Phase 3: Code Deduplication (DRY)

### 3.1 Extract shared typewriter animation utility
**Duplicated in:** `ReadView.swift`, `WriteView.swift`, `BonusReadView.swift`
- `wordChunks(from:)` and `enqueueTypewriter()` are copy-pasted verbatim (~60 lines each)
- **Fix:** Create `Utilities/TypewriterAnimator.swift` as an `@Observable @MainActor` class or a shared static utility. Views can hold it via `@State`.
- Contains: `pendingChunks`, `displayedText`, `revealTask`, `enqueueTypewriter()`, `wordChunks(from:)`

### 3.2 Extract shared `markdownAttributedString` helper
**Duplicated in:** `ReadView.swift`, `BonusReadView.swift`, `ReadingLogView.swift`
- Identical 6-line static method in 3 files
- **Fix:** Create a `String` extension or free function in `Utilities/MarkdownHelper.swift`

### 3.3 Extract shared `statColumn` component
**Duplicated in:** `DoneView.swift` (line ~113), `StatsView.swift` (line ~46)
- Identical stat column helper
- **Fix:** Create `Views/Components/StatColumnView.swift` as a reusable component

### 3.4 Consolidate APIClient SSE streaming methods
**File:** `Services/APIClient.swift` (lines ~141-322 vs ~334-509)
- `streamReading()` and `streamPrompt()` are ~180 lines each, nearly identical
- **Fix:** Create a generic `private func streamSSE<T>(path:parser:) -> AsyncThrowingStream<T, Error>` and refactor both methods to call it with different parsers
- This is the largest DRY violation (~360 lines of duplicated logic)

### 3.5 Consolidate APIClient REST methods
**File:** `Services/APIClient.swift` (lines ~561-651)
- `get()`, `post()`, `put()` each copy the same 401-retry boilerplate (~30 lines each)
- **Fix:** Create a single `private func performRequest<T>(method:path:body:) -> T` with shared retry logic

## Phase 6: Minor Improvements

### 6.1 Cache `ISO8601DateFormatter` in SyncService
**File:** `Services/SyncService.swift` (line ~38)
- Creates a new `ISO8601DateFormatter()` per entry in `.map`
- **Fix:** Use a `static let` shared formatter

### 6.2 Improve error logging for `try? modelContext.save()`
**Files:** Throughout views and services (15+ occurrences)
- Silent `try?` swallows save errors
- **Fix:** Replace with `do { try modelContext.save() } catch { print("Context save failed: \(error)") }` for at least the critical paths (profile creation, writing completion)

### 6.3 Guard against overlapping sync tasks
**File:** `Views/RootView.swift` (lines ~93-99)
- Unstructured `Task` in `onChange(of: scenePhase)` with no cancellation
- **Fix:** Store the sync task in `@State` and cancel the previous one before starting a new one
