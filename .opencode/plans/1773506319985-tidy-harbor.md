# Fix: QStash Workflow signature verification failure in production

## Root Cause

The error occurs on the **Vercel backend**, not in the iOS app. The iOS app has no env variable issue.

**What happens on production (Vercel):**

1. iOS (TestFlight) POST → `/api/readings/generate/stream` with Bearer token
2. Route handler runs pre-processing (auth, lock, payload resolution)
3. Reaches the production path (line 302-317 in readings, 245-259 in prompts):
   - `NODE_ENV !== "production"` is **false** on Vercel → dev path skipped
   - Calls `workflowPOST(enrichedRequest)` directly
4. `serve()` from `@upstash/workflow` sees `QSTASH_CURRENT_SIGNING_KEY`/`QSTASH_NEXT_SIGNING_KEY` are set → tries to verify `upstash-signature` header
5. **Fails** because the request came from iOS, not QStash — no signature present

**Why it works locally:** `NODE_ENV !== "production"` is **true** → the dev path fires `runReadingGeneration()` directly in-process, never calling `workflowPOST()`.

## Fix

Two changes needed:

### 1. Use QStash `Client.publish()` for the initial production trigger

Instead of calling `workflowPOST(enrichedRequest)` directly (which triggers signature verification), publish the payload to QStash so it calls back to the endpoint with a proper signature.

**Files to modify:**
- `apps/web/app/api/readings/generate/stream/route.ts` (lines 310-317)
- `apps/web/app/api/prompts/generate/stream/route.ts` (lines 253-259)

Replace:
```typescript
const enrichedRequest = new Request(request.url, {
  method: request.method,
  headers: request.headers,
  body: JSON.stringify(payload),
});
return workflowPOST(enrichedRequest);
```

With:
```typescript
import { Client } from "@upstash/qstash";

const qstash = new Client({ token: process.env.QSTASH_TOKEN! });

// ... in the production path:
const baseUrl = request.headers.get("host")?.includes("localhost")
  ? `http://${request.headers.get("host")}`
  : `https://${request.headers.get("host")}`;

await qstash.publishJSON({
  url: `${baseUrl}/api/readings/generate/stream`,
  body: payload,
  headers: {
    Authorization: request.headers.get("Authorization") || "",
  },
});
return Response.json({ ok: true, mode: "started" }, { status: 202 });
```

This makes QStash the intermediary: QStash calls back to the same endpoint with a valid `upstash-signature`. The existing `if (hasQstashSignature)` branch then passes the signed request to `workflowPOST()`, which verifies successfully.

### 2. Update proxy to allow QStash-signed requests

When QStash orchestrates workflow step callbacks (after the initial trigger), the SDK publishes subsequent steps to QStash. These callbacks may or may not preserve the original `Authorization` header depending on SDK behavior. To be safe, allow requests with a valid `upstash-signature` header through the proxy.

**File:** `apps/web/proxy.ts`

```typescript
export function proxy(request: NextRequest) {
  const auth = request.headers.get("Authorization");
  const hasQstashSignature = request.headers.get("upstash-signature");

  if (!auth?.startsWith("Bearer ") && !hasQstashSignature) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
  return NextResponse.next();
}
```

This is safe because: QStash signature is verified cryptographically inside `workflowPOST()`. The proxy is just an early-rejection layer — the real auth check happens in the route handler.

### 3. Add `@upstash/qstash` as a direct dependency

Currently a transitive dependency via `@upstash/workflow`. Add it explicitly:

```
bun add @upstash/qstash --cwd apps/web
```

## Production request flow after fix

```
iOS POST (Bearer) → Route handler (auth, lock, payload) → qstash.publishJSON(url, payload)
                                                            ↓
                              QStash calls POST with upstash-signature + Authorization
                                                            ↓
                              Proxy allows (has signature) → hasQstashSignature=true
                                                            ↓
                              workflowPOST(request) → verify signature ✓ → initial call
                                                            ↓
                              SDK publishes step 1 to QStash
                                                            ↓
                              QStash calls POST with signature → workflowPOST → step 1
                                                            ↓
                              workflow.run("reading-generation") → runReadingGeneration()
```
