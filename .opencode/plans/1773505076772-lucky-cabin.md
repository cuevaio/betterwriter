# Plan: Fix App Store Connect Icon (Grey Placeholder)

## Problem

The app icon shows as a grey placeholder in App Store Connect's Distribution dashboard,
even though the icon appears correctly in the iOS Simulator.

**Root cause:** App Store Connect requires the 1024×1024 marketing icon to be uploaded
**separately** as part of the App Store listing metadata. The current Fastlane `beta` lane
only calls `pilot` (TestFlight upload), which uploads the binary (IPA). The binary contains
the icon (for device display), but the **App Store storefront listing** needs the icon
submitted via `deliver` or the App Store Connect web UI.

## Two Options

### Option A — Manual upload via App Store Connect UI (quickest fix, one-time)

1. Go to App Store Connect → your app → App Information
2. Scroll to **App Icon** section
3. Upload `apps/ios/betterwriter/betterwriter/Assets.xcassets/AppIcon.appiconset/AppIcon.png`
   (1024×1024, no alpha channel — PNG alpha must be stripped first)
4. Save

**Caveat:** PNG must have no alpha channel. The current `AppIcon.png` may have an alpha
channel (needs verification). If it does, strip alpha: `convert AppIcon.png -background white -alpha remove AppIcon-store.png` (ImageMagick).

### Option B — Automate via `deliver` in the Fastlane `beta` lane (recommended)

Add a `deliver` step after `pilot` in `apps/ios/fastlane/Fastfile` so every release
keeps the listing icon up to date automatically.

Steps:
1. Create `apps/ios/fastlane/metadata/` directory with the required icon file.
2. Append `deliver` call to the `beta` lane (metadata only, no screenshots required).

## Recommended Approach

**Option B only** — automate via `deliver` in the Fastlane `beta` lane.

## Files to Modify (Option B)

- `apps/ios/fastlane/Fastfile` — add `deliver` action after `pilot`

New `deliver` block to append inside the `beta` lane, after the `pilot(...)` call:

```ruby
# 7. Upload App Store listing icon / metadata
deliver(
  api_key: api_key,
  app_identifier: "cuevaio.betterwriter",
  skip_screenshots: true,
  skip_metadata: true,        # set to false once metadata/ folder exists
  app_icon: "./fastlane/metadata/AppIcon-store.png",
  overwrite_screenshots: false,
  submit_for_review: false,
  force: true
)
```

And add the no-alpha icon at `apps/ios/fastlane/metadata/AppIcon-store.png`
(derived from `AppIcon.png` with alpha stripped).

## Verification

1. Strip alpha from the PNG and upload manually via App Store Connect UI.
2. Refresh the App Store Connect Distribution page — the icon should appear immediately
   after saving (no new build needed).
3. For Option B: run `bundle exec fastlane beta` locally (or trigger CI) and confirm
   `deliver` succeeds and the icon is reflected in App Store Connect.

## Notes

- The asset catalog setup (`Contents.json`) is correct and does not need changes.
- The icon IS embedded in the IPA correctly (that's why the simulator shows it).
- This is purely an App Store listing metadata issue.
- `AppIcon.png` is already RGB (no alpha channel) — can be used as-is. No stripping needed.
- The icon file to add at `apps/ios/fastlane/metadata/AppIcon-store.png` is simply a copy
  of `apps/ios/betterwriter/betterwriter/Assets.xcassets/AppIcon.appiconset/AppIcon.png`.
