# Plan: Fix App Store Export Compliance (Encryption Declaration)

## Context

Apple's App Store Connect is showing "Missing Compliance" on TestFlight builds and prompting for App Encryption Documentation. The app must declare whether it uses non-exempt encryption.

## Analysis

Audit of `apps/ios` confirms the app uses **only OS-level security mechanisms**:
- Keychain (`Security` framework) for JWT token and device UUID storage
- HTTPS/TLS via `URLSession` + Apple ATS (standard iOS networking)

No custom encryption, no CryptoKit, no CommonCrypto, no third-party crypto libraries.

**Correct answer to Apple's dialog:** "None of the algorithms mentioned above"

## Fix

Add `ITSAppUsesNonExemptEncryption = NO` to the Info.plist source file. This tells Apple the app is exempt from export compliance requirements and suppresses the Missing Compliance prompt for all future builds automatically.

### File to modify

`apps/ios/betterwriter/betterwriter-Info.plist`

### Change

Add inside the root `<dict>`:

```xml
<key>ITSAppUsesNonExemptEncryption</key>
<false/>
```

The resulting plist will have three top-level keys:
1. `NSAppTransportSecurity` (already present)
2. `UILaunchScreen` (already present)
3. `ITSAppUsesNonExemptEncryption` = `false` (new)

## Verification

1. Build the app in Xcode (or via `bundle exec fastlane build` in `apps/ios`)
2. Upload a new TestFlight build
3. The "Missing Compliance" warning should no longer appear on the build in App Store Connect
4. The encryption documentation dialog should not prompt for future builds

## Notes

- No Swift code changes required
- This is the standard approach for apps that only use HTTPS and Keychain (Apple-managed encryption)
- The `ITSAppUsesNonExemptEncryption = NO` key is recognized by App Store Connect and auto-resolves compliance
