# Plan: Count Bonus Readings and Free Writes in Word Totals

## Goal
Bonus reading sessions (`isBonusReading = true`) and free write sessions (`isFreeWrite = true`) should count toward the total words read and written statistics, respectively.

## Current Behavior
- `DayEngine.totalWordsRead` excludes bonus readings (`!$0.isBonusReading` filter)
- `DayEngine.totalWordsWritten` already includes free writes (no filter — sums all entries)
- `CalendarGridView.entryByDate` excludes bonus readings from the activity chart

**Key finding:** Free writes are already counted in `totalWordsWritten`. Only bonus readings need to be included.

## Changes Required

### 1. `apps/ios/betterwriter/betterwriter/Services/DayEngine.swift` (line 166–169)

Remove the `!$0.isBonusReading` filter from `totalWordsRead`:

```swift
// BEFORE
static func totalWordsRead(entries: [DayEntry]) -> Int {
    entries.filter { $0.readingCompleted && !$0.isBonusReading }
        .reduce(0) { $0 + $1.readingWordCount }
}

// AFTER
static func totalWordsRead(entries: [DayEntry]) -> Int {
    entries.filter { $0.readingCompleted }
        .reduce(0) { $0 + $1.readingWordCount }
}
```

### 2. `apps/ios/betterwriter/betterwriter/Views/Components/CalendarGridView.swift` (line 19)

Remove `!entry.isBonusReading` from the `entryByDate` dictionary builder so bonus readings contribute to the activity chart heat-map:

```swift
// BEFORE
for entry in entries where !entry.isBonusReading && !entry.skipped {

// AFTER
for entry in entries where !entry.skipped {
```

However, the current `entryByDate` uses only one entry per calendar date (picks highest word count). Bonus readings are synthetic entries with a `dayIndex` in the 100k–199k range, so they won't naturally merge with the normal day entry. We need to **aggregate** all entries per date instead of replacing with the highest one.

Updated `entryByDate` approach — sum reading + writing words per calendar date and store in a simple `[DateComponents: Int]` map (or keep `DayEntry` logic but accumulate words):

```swift
// Simplest fix: sum all non-skipped entries per calendar day
private var wordsByDate: [DateComponents: Int] {
    var map: [DateComponents: Int] = [:]
    for entry in entries where !entry.skipped {
        let comps = calendar.dateComponents([.year, .month, .day], from: entry.calendarDate)
        map[comps, default: 0] += entry.readingWordCount + entry.writingWordCount
    }
    return map
}
```

Then update `cellColor` and `activityLevel` to use `Int` instead of `DayEntry?`:

```swift
private func cellColor(for date: Date?) -> Color {
    guard let date = date else { return Color.clear }
    let comps = calendar.dateComponents([.year, .month, .day], from: date)
    let level = activityLevel(for: lookup[comps])
    return colorForLevel(level)
}

private func activityLevel(for wordCount: Int?) -> Int {
    let total = wordCount ?? 0
    if total == 0 { return 0 }
    if total < 100 { return 1 }
    if total < 300 { return 2 }
    if total < 600 { return 3 }
    return 4
}
```

The `ActivityChartGrid` struct's `lookup` property type changes from `[DateComponents: DayEntry]` to `[DateComponents: Int]`.
